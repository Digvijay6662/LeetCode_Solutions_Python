You are given an integer numberOfUsers representing the total number of users and an array events of size n x 3.

Each events[i] can be either of the following two types:

Message Event: ["MESSAGE", "timestampi", "mentions_stringi"]
This event indicates that a set of users was mentioned in a message at timestampi.
The mentions_stringi string can contain one of the following tokens:
id<number>: where <number> is an integer in range [0,numberOfUsers - 1]. There can be multiple ids separated by a single whitespace and may contain duplicates. This can mention even the offline users.
ALL: mentions all users.
HERE: mentions all online users.
Offline Event: ["OFFLINE", "timestampi", "idi"]
This event indicates that the user idi had become offline at timestampi for 60 time units. The user will automatically be online again at time timestampi + 60.
Return an array mentions where mentions[i] represents the number of mentions the user with id i has across all MESSAGE events.

All users are initially online, and if a user goes offline or comes back online, their status change is processed before handling any message event that occurs at the same timestamp.

Note that a user can be mentioned multiple times in a single message event, and each mention should be counted separately.

 

Example 1:

Input: numberOfUsers = 2, events = [["MESSAGE","10","id1 id0"],["OFFLINE","11","0"],["MESSAGE","71","HERE"]]

Output: [2,2]

Explanation:

Initially, all users are online.

At timestamp 10, id1 and id0 are mentioned. mentions = [1,1]

At timestamp 11, id0 goes offline.

At timestamp 71, id0 comes back online and "HERE" is mentioned. mentions = [2,2]

Example 2:

Input: numberOfUsers = 2, events = [["MESSAGE","10","id1 id0"],["OFFLINE","11","0"],["MESSAGE","12","ALL"]]

Output: [2,2]

Explanation:

Initially, all users are online.

At timestamp 10, id1 and id0 are mentioned. mentions = [1,1]

At timestamp 11, id0 goes offline.

At timestamp 12, "ALL" is mentioned. This includes offline users, so both id0 and id1 are mentioned. mentions = [2,2]

Example 3:

Input: numberOfUsers = 2, events = [["OFFLINE","10","0"],["MESSAGE","12","HERE"]]

Output: [0,1]

Explanation:

Initially, all users are online.

At timestamp 10, id0 goes offline.

At timestamp 12, "HERE" is mentioned. Because id0 is still offline, they will not be mentioned. mentions = [0,1]

 

Constraints:

1 <= numberOfUsers <= 100
1 <= events.length <= 100
events[i].length == 3
events[i][0] will be one of MESSAGE or OFFLINE.
1 <= int(events[i][1]) <= 105
The number of id<number> mentions in any "MESSAGE" event is between 1 and 100.
0 <= <number> <= numberOfUsers - 1
It is guaranteed that the user id referenced in the OFFLINE event is online at the time the event occurs.


Solution 1:

class Solution:
    def countMentions(self, numberOfUsers: int, events: List[List[str]]) -> List[int]:
        # Comparator function to sort events:
        # 1. Sort primarily by timestamp (ascending).
        # 2. If two events have the same timestamp, process OFFLINE before MESSAGE.
        def comparator(event1, event2):
            time1, time2 = int(event1[1]), int(event2[1])
            if time1 != time2:
                return time1 - time2
            # Ensure OFFLINE is processed before MESSAGE at the same time
            if event1[0] == "OFFLINE" and event2[0] == "MESSAGE":
                return -1
            if event1[0] == "MESSAGE" and event2[0] == "OFFLINE":
                return 1
            return 0

        # Sort events using the custom comparator
        events.sort(key=cmp_to_key(comparator))

        # Dictionary to track users who are offline.
        # Key = user_id, Value = timestamp when they went offline.
        offline_until = {}

        # Array to store mention counts for each user.
        # Index = user_id, Value = total mentions.
        mention_counts = [0] * numberOfUsers

        # Process each event in chronological order
        for event in events:
            current_time = int(event[1])  # Current event timestamp

            if event[0] == "MESSAGE":
                message_content = event[2]  # The message string (ALL, HERE, or explicit ids)
                if message_content == "ALL":
                    # ALL mentions every user, regardless of online/offline status
                    for user_id in range(numberOfUsers):
                        mention_counts[user_id] += 1
                elif message_content == "HERE":
                    # HERE mentions only currently online users
                    for user_id in range(numberOfUsers):
                        if user_id not in offline_until:
                            # User is online → count mention
                            mention_counts[user_id] += 1
                        else:
                            # User is offline → check if their offline period has expired
                            offline_time = offline_until[user_id]
                            if current_time >= offline_time + 60:
                                # User comes back online automatically after 60 units
                                del offline_until[user_id]
                                mention_counts[user_id] += 1
                else:
                    # Explicit mentions like "id0 id1 id1"
                    ids = message_content.split()
                    for id_str in ids:
                        # Extract user_id from "idX" format
                        user_id = int(id_str[2:])
                        mention_counts[user_id] += 1
            else:  # OFFLINE event
                # OFFLINE events provide plain user_id (e.g., "0")
                user_id = int(event[2])
                # Mark user as offline starting at current_time
                offline_until[user_id] = current_time

        # Return the final mention counts for all users
        return mention_counts
