You are given an integer n, representing n nodes numbered from 0 to n - 1 and a list of edges, where edges[i] = [ui, vi, si, musti]:

ui and vi indicates an undirected edge between nodes ui and vi.
si is the strength of the edge.
musti is an integer (0 or 1). If musti == 1, the edge must be included in the spanning tree. These edges cannot be upgraded.
You are also given an integer k, the maximum number of upgrades you can perform. Each upgrade doubles the strength of an edge, and each eligible edge (with musti == 0) can be upgraded at most once.

The stability of a spanning tree is defined as the minimum strength score among all edges included in it.

Return the maximum possible stability of any valid spanning tree. If it is impossible to connect all nodes, return -1.

Note: A spanning tree of a graph with n nodes is a subset of the edges that connects all nodes together (i.e. the graph is connected) without forming any cycles, and uses exactly n - 1 edges.

 

Example 1:

Input: n = 3, edges = [[0,1,2,1],[1,2,3,0]], k = 1

Output: 2

Explanation:

Edge [0,1] with strength = 2 must be included in the spanning tree.
Edge [1,2] is optional and can be upgraded from 3 to 6 using one upgrade.
The resulting spanning tree includes these two edges with strengths 2 and 6.
The minimum strength in the spanning tree is 2, which is the maximum possible stability.
Example 2:

Input: n = 3, edges = [[0,1,4,0],[1,2,3,0],[0,2,1,0]], k = 2

Output: 6

Explanation:

Since all edges are optional and up to k = 2 upgrades are allowed.
Upgrade edges [0,1] from 4 to 8 and [1,2] from 3 to 6.
The resulting spanning tree includes these two edges with strengths 8 and 6.
The minimum strength in the tree is 6, which is the maximum possible stability.
Example 3:

Input: n = 3, edges = [[0,1,1,1],[1,2,1,1],[2,0,1,1]], k = 0

Output: -1

Explanation:

All edges are mandatory and form a cycle, which violates the spanning tree property of acyclicity. Thus, the answer is -1.
 

Constraints:

2 <= n <= 105
1 <= edges.length <= 105
edges[i] = [ui, vi, si, musti]
0 <= ui, vi < n
ui != vi
1 <= si <= 105
musti is either 0 or 1.
0 <= k <= n
There are no duplicate edges.


Solution 1:

# Union-Find (Disjoint Set Union) data structure
# Used to efficiently check connectivity and detect cycles
class UnionFind:
    def __init__(self, n: int):
        # Each node starts as its own parent (self loop)
        self.parent = [i for i in range(n)]
        # Track the size of each connected component
        self.size = [1] * n

    def find(self, node: int) -> int:
        # Path compression: flatten the tree for efficiency
        while node != self.parent[node]:
            self.parent[node] = self.parent[self.parent[node]]
            node = self.parent[node]
        return node

    def union(self, node1: int, node2: int) -> bool:
        # Find roots of both nodes
        root1, root2 = self.find(node1), self.find(node2)
        if root1 == root2:
            # Already connected → union not possible
            return False

        # Union by size: attach smaller tree under larger one
        if self.size[root1] > self.size[root2]:
            self.parent[root2] = root1
            self.size[root1] += self.size[root2]
        else:
            self.parent[root1] = root2
            self.size[root2] += self.size[root1]

        return True

class Solution:
    def maxStability(self, n: int, edges: List[List[int]], k: int) -> int:
        # Helper function: check if we can build a spanning tree
        # with minimum edge strength >= targetStrength
        def canBuildSpanningTree(targetStrength):
            upgradesLeft = k
            uf = UnionFind(n)
            upgradeCandidates = []  # edges that can reach target if upgraded

            for u, v, strength, mustInclude in edges:
                if mustInclude == 1:  # mandatory edge
                    if strength < targetStrength:
                        # Mandatory edge too weak → fail immediately
                        return False
                    uf.union(u, v)
                else:  # optional edge
                    if strength >= targetStrength:
                        # Already strong enough → use directly
                        uf.union(u, v)
                    elif 2 * strength >= targetStrength:
                        # Can reach target if upgraded → store for later
                        upgradeCandidates.append([u, v])

            # Try to use upgrades on candidate edges to connect components
            for u, v in upgradeCandidates:
                if uf.find(u) != uf.find(v) and upgradesLeft > 0:
                    uf.union(u, v)
                    upgradesLeft -= 1

            # Final check: ensure all nodes are connected
            root = uf.find(0)
            for i in range(1, n):
                if root != uf.find(i):
                    return False
            return True

        # Step 1: Check mandatory edges for cycles
        uf = UnionFind(n)
        minStrength, maxStrength = float('inf'), float('-inf')

        for u, v, strength, mustInclude in edges:
            if mustInclude == 1:
                if uf.find(u) == uf.find(v):
                    # Cycle detected among mandatory edges → invalid
                    return -1
                uf.union(u, v)
            # Update bounds: minimum strength and maximum possible strength (after upgrade)
            minStrength = min(minStrength, strength)
            maxStrength = max(maxStrength, 2 * strength)

        # Step 2: Binary search for maximum possible stability
        left, right = minStrength, maxStrength
        bestStability = -1

        while left <= right:
            mid = left + (right - left) // 2  # candidate stability
            if canBuildSpanningTree(mid):
                bestStability = mid          # feasible → try higher
                left = mid + 1
            else:
                right = mid - 1              # not feasible → try lower

        return bestStability
