You are given an integer array nums of length n and an integer k.

You need to choose exactly k non-empty subarrays nums[l..r] of nums. Subarrays may overlap, and the exact same subarray (same l and r) can be chosen more than once.

The value of a subarray nums[l..r] is defined as: max(nums[l..r]) - min(nums[l..r]).

The total value is the sum of the values of all chosen subarrays.

Return the maximum possible total value you can achieve.

 

Example 1:

Input: nums = [1,3,2], k = 2

Output: 4

Explanation:

One optimal approach is:

Choose nums[0..1] = [1, 3]. The maximum is 3 and the minimum is 1, giving a value of 3 - 1 = 2.
Choose nums[0..2] = [1, 3, 2]. The maximum is still 3 and the minimum is still 1, so the value is also 3 - 1 = 2.
Adding these gives 2 + 2 = 4.

Example 2:

Input: nums = [4,2,5,1], k = 3

Output: 12

Explanation:

One optimal approach is:

Choose nums[0..3] = [4, 2, 5, 1]. The maximum is 5 and the minimum is 1, giving a value of 5 - 1 = 4.
Choose nums[0..3] = [4, 2, 5, 1]. The maximum is 5 and the minimum is 1, so the value is also 4.
Choose nums[2..3] = [5, 1]. The maximum is 5 and the minimum is 1, so the value is again 4.
Adding these gives 4 + 4 + 4 = 12.

 

Constraints:

1 <= n == nums.length <= 5 * 10​​​​​​​4
0 <= nums[i] <= 109
1 <= k <= 105


Solution 1:

class Solution:
    def maxTotalValue(self, nums: List[int], k: int) -> int:
        # ── GREEDY INSIGHT ──
        # value(subarray) = max(subarray) - min(subarray)
        # No subarray can exceed global_max - global_min because:
        #   max(any subarray) ≤ global_max
        #   min(any subarray) ≥ global_min
        # So (global_max - global_min) is an absolute ceiling per pick.
        # Since subarrays can be reused, just pick the best one k times:
        #   answer = (global_max - global_min) × k

        # ── SEED WITH FIRST ELEMENT ──
        # Both trackers start at nums[0] so the very first comparison
        # in the loop is meaningful (comparing against a real value,
        # not a placeholder like float('inf') or float('-inf')).
        # This also correctly handles the edge case of n = 1.
        global_max = global_min = nums[0]

        # ── SINGLE PASS: find global_max and global_min simultaneously ──
        # Start from index 1 since index 0 already seeds both trackers above.
        # One pass = half the memory traffic vs calling max() then min()
        # separately (each of which does a full independent pass).
        for num in nums[1:]:
            if num > global_max:
                # Found a new highest value → update ceiling
                global_max = num

            elif num < global_min:
                # Found a new lowest value → update floor
                #
                # ── WHY elif AND NOT if ──
                # A single number cannot simultaneously be strictly greater
                # than global_max AND strictly less than global_min.
                # Using elif avoids a redundant second comparison on the
                # common case where num > global_max (already handled above),
                # saving one comparison per such iteration.
                global_min = num

            # If neither branch fires: global_min ≤ num ≤ global_max
            # → num is already within the known range → no update needed

        # ── FINAL CALCULATION ──
        # global_max - global_min = best achievable value for one subarray
        #                           (the whole array always contains both)
        # Multiply by k because we can reuse the same best subarray k times
        return (global_max - global_min) * k
