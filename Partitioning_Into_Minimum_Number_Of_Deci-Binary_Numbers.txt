A decimal number is called deci-binary if each of its digits is either 0 or 1 without any leading zeros. For example, 101 and 1100 are deci-binary, while 112 and 3001 are not.

Given a string n that represents a positive decimal integer, return the minimum number of positive deci-binary numbers needed so that they sum up to n.

 

Example 1:

Input: n = "32"
Output: 3
Explanation: 10 + 11 + 11 = 32
Example 2:

Input: n = "82734"
Output: 8
Example 3:

Input: n = "27346209830709182346"
Output: 9
 

Constraints:

1 <= n.length <= 105
n consists of only digits.
n does not contain any leading zeros and represents a positive integer.


Solution 1:

class Solution:
    def minPartitions(self, n: str) -> int:
        # The problem asks for the minimum number of deci-binary numbers
        # (numbers made up of only digits 0 or 1) that sum up to the given number n.
        
        # Key observation:
        # Each digit in n requires at least that many deci-binary numbers.
        # For example, if a digit is '8', we need at least 8 deci-binary numbers
        # because each deci-binary number can contribute at most '1' in that digit place.
        
        # Therefore, the minimum number of deci-binary numbers required
        # is simply the maximum digit in n.
        
        # max(n) finds the largest character (digit) in the string n.
        # int(max(n)) converts that character back into an integer.
        return int(max(n))
