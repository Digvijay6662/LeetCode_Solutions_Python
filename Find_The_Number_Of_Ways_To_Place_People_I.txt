3025. Find the Number of Ways to Place People I
Solved
Medium
Companies
Hint
You are given a 2D array points of size n x 2 representing integer coordinates of some points on a 2D-plane, where points[i] = [xi, yi].

We define the right direction as positive x-axis (increasing x-coordinate) and the left direction as negative x-axis (decreasing x-coordinate). Similarly, we define the up direction as positive y-axis (increasing y-coordinate) and the down direction as negative y-axis (decreasing y-coordinate)

You have to place n people, including Chisato and Takina, at these points such that there is exactly one person at every point. Chisato wants to be alone with Takina, so Chisato will build a rectangular fence with Chisato's position as the upper left corner and Takina's position as the lower right corner of the fence (Note that the fence might not enclose any area, i.e. it can be a line). If any person other than Chisato and Takina is either inside the fence or on the fence, Chisato will be sad.

Return the number of pairs of points where you can place Chisato and Takina, such that Chisato does not become sad on building the fence.

Note that Chisato can only build a fence with Chisato's position as the upper left corner, and Takina's position as the lower right corner. For example, Chisato cannot build either of the fences in the picture below with four corners (1, 1), (1, 3), (3, 1), and (3, 3), because:

With Chisato at (3, 3) and Takina at (1, 1), Chisato's position is not the upper left corner and Takina's position is not the lower right corner of the fence.
With Chisato at (1, 3) and Takina at (1, 1), Takina's position is not the lower right corner of the fence.

 

Example 1:


Input: points = [[1,1],[2,2],[3,3]]
Output: 0
Explanation: There is no way to place Chisato and Takina such that Chisato can build a fence with Chisato's position as the upper left corner and Takina's position as the lower right corner. Hence we return 0. 
Example 2:


Input: points = [[6,2],[4,4],[2,6]]
Output: 2
Explanation: There are two ways to place Chisato and Takina such that Chisato will not be sad:
- Place Chisato at (4, 4) and Takina at (6, 2).
- Place Chisato at (2, 6) and Takina at (4, 4).
You cannot place Chisato at (2, 6) and Takina at (6, 2) because the person at (4, 4) will be inside the fence.
Example 3:


Input: points = [[3,1],[1,3],[1,1]]
Output: 2
Explanation: There are two ways to place Chisato and Takina such that Chisato will not be sad:
- Place Chisato at (1, 1) and Takina at (3, 1).
- Place Chisato at (1, 3) and Takina at (1, 1).
You cannot place Chisato at (1, 3) and Takina at (3, 1) because the person at (1, 1) will be on the fence.
Note that it does not matter if the fence encloses any area, the first and second fences in the image are valid.
 

Constraints:

2 <= n <= 50
points[i].length == 2
0 <= points[i][0], points[i][1] <= 50
All points[i] are distinct.


Solution 1:

class Solution:
    def numberOfPairs(self, points: List[List[int]]) -> int:
        n = len(points)
        result = 0
        
        for i in range(n):
            top_left = points[i]
            for j in range(n):
                if i == j:
                    continue

                bottom_right = points[j]
                if top_left[0] > bottom_right[0] or top_left[1] < bottom_right[1]:
                    continue

                flag = True
                for k in range(n):
                    if i == k or j == k:
                        continue

                    if top_left[0] <= points[k][0] <= bottom_right[0] and top_left[1] >= points[k][1] >= bottom_right[1]:
                        flag = False
                        break

                if flag:
                    result += 1

        return result

class Solution:
    def numberOfPairs(self, points: List[List[int]]) -> int:
        n = len(points)  # Total number of points
        result = 0       # Counter for valid (A, B) pairs

        # Sort points by:
        # 1. x-coordinate ascending
        # 2. y-coordinate descending (so higher y comes first if x is the same)
        # This ordering helps ensure that when we scan to the right,
        # all points have x >= current point's x, and ties are handled correctly.
        sorted_points = sorted(points, key=lambda x: (x[0], -x[1]))

        # Outer loop: pick each point as candidate A (upper-left point)
        for i in range(n):
            max_y = -1  # Tracks the highest y encountered so far among valid B's
                        # This helps ensure no point lies inside the rectangle

            # Inner loop: pick each later point as candidate B (lower-right point)
            for j in range(i + 1, n):
                # Skip if B is above A (A must be strictly above B)
                if sorted_points[i][1] < sorted_points[j][1]:
                    continue

                # If B's y is higher than any previously seen B's y in this scan,
                # it means there is no point between A and B that blocks the rectangle.
                if sorted_points[j][1] > max_y:
                    max_y = sorted_points[j][1]  # Update the highest y seen so far
                    result += 1                  # Count this as a valid pair

        return result  # Return the total number of valid pairs

class Solution:
    def numberOfPairs(self, points: List[List[int]]) -> int:
        # Custom comparator function for sorting points
        # Sort primarily by x-coordinate in ascending order
        # If x is the same, sort by y-coordinate in descending order
        def comparator(a, b):
            if a[0] != b[0]:          # Compare x-coordinates
                if a[0] < b[0]:
                    return -1         # a comes before b
                else:
                    return 1          # b comes before a
            else:                     # If x is the same
                if a[1] < b[1]:
                    return 1          # Higher y comes first
                else:
                    return -1

        n = len(points)  # Number of points
        result = 0       # Counter for valid (A, B) pairs

        # Sort points using the custom comparator
        # After sorting:
        # - Points are ordered left to right by x
        # - For same x, higher y comes first
        sorted_points = sorted(points, key=cmp_to_key(comparator))

        # Outer loop: choose each point as candidate A (upper-left point)
        for i in range(n):
            max_y = -1  # Tracks the highest y encountered so far among valid B's
                        # Ensures no point lies inside the rectangle between A and B

            # Inner loop: choose each later point as candidate B (lower-right point)
            for j in range(i + 1, n):
                # Skip if B is above A (A must be strictly above B)
                if sorted_points[i][1] < sorted_points[j][1]:
                    continue

                # If B's y is higher than any previously seen B's y in this scan,
                # it means no intermediate point blocks the rectangle
                if sorted_points[j][1] > max_y:
                    max_y = sorted_points[j][1]  # Update the highest y seen so far
                    result += 1                  # Count this as a valid pair

        return result  # Return the total number of valid pairs
